## 1. 逃逸分析

编译器决定变量分配在栈还是堆：

- **栈分配**：函数返回后自动回收，零成本
- **堆分配**：需要 GC 回收，有额外开销

```bash
go build -gcflags="-m" main.go
```

## 2. 常见逃逸场景

### 2.1 返回局部变量指针

```go
func newInt() *int {
    x := 42
    return &x  // x 逃逸到堆
}
```

### 2.2 interface 类型

```go
func print(v interface{}) {
    fmt.Println(v)  // v 逃逸
}
```

### 2.3 闭包捕获

```go
func counter() func() int {
    x := 0
    return func() int {
        x++  // x 逃逸
        return x
    }
}
```

### 2.4 切片扩容

```go
func grow() []int {
    s := make([]int, 0)
    for i := 0; i < 1000; i++ {
        s = append(s, i)  // 可能多次扩容，逃逸
    }
    return s
}
```

## 3. 优化建议

- 避免返回局部变量指针
- 使用值类型而非指针（小对象）
- 预分配切片大小
- 使用 sync.Pool 复用对象
