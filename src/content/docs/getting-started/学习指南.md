## 1. 学习节奏

### 1.1 时间规划

| 阶段   | 每日投入 | 持续时间 | 目标                         |
| ------ | -------- | -------- | ---------------------------- |
| 入门期 | 1-2小时  | 2-4周    | 掌握基础工具和一门语言语法   |
| 成长期 | 2-3小时  | 2-3月    | 能独立完成小项目             |
| 进阶期 | 2-4小时  | 3-6月    | 理解底层原理, 能解决复杂问题 |

### 1.2 学习原则

- **每天坚持** - 30分钟也好过周末突击5小时
- **动手为主** - 看一遍不如敲一遍, 敲一遍不如改一遍
- **不求甚解** - 第一遍不必完全理解, 先用起来再深入
- **记录笔记** - 用Markdown记录学习过程, 这本身就是练习

## 2. 常见问题

### 2.1 "我该学哪门语言?"

取决于你的目标:

| 目标     | 推荐语言   | 原因               |
| -------- | ---------- | ------------------ |
| Web前端  | JavaScript | 前端唯一语言       |
| 数据分析 | Python     | 生态完善, 语法简洁 |
| 系统编程 | C/C++      | 底层控制, 性能极致 |
| 企业开发 | Java       | 岗位多, 生态成熟   |
| 游戏开发 | C#/C++     | Unity/Unreal引擎   |

### 2.2 "学不会怎么办?"

- 降低难度: 回到上一个理解的知识点
- 换个方式: 看视频教程替代纯文字
- 找人讨论: 在GitHub Issues或社区提问
- 休息一下: 睡一觉, 大脑会在后台处理

### 2.3 "学了就忘怎么办?"

遗忘是正常的. 应对方法:

1. **间隔复习** - 第1天、第3天、第7天各回顾一次
2. **费曼技巧** - 用自己的话解释给别人听
3. **项目驱动** - 做一个小项目, 把知识点串起来
4. **标记进度** - 用本站进度系统标记已读文档

## 3. 心态建议

- 编程是一项技能, 不是天赋. 任何人都可以通过练习掌握
- 报错是正常的, 专业开发者每天也在调试错误
- 不要和别人比速度, 按自己的节奏前进
- 完成比完美重要, 先让代码跑起来再优化
- 遇到困难时, 记住: 你已经比昨天的自己进步了
