## 这是什么

FANDEX 是一条循序渐进的学习路径。无论你是刚接触编程、跨专业转行、还是想系统梳理知识体系，这里都为你提供了一条清晰的前进方向。

51 个知识模块、2000 余篇文档，从"如何安装一个编辑器"到"如何训练一个大语言模型"，按照渐进式学习原则编排。每篇文档聚焦一个知识点，包含概念讲解、代码示例和关键术语。

## 从哪里开始

### 完全零基础

如果你从未写过代码，按以下顺序开始：

1. **开发环境搭建** -- 安装编辑器和命令行工具，让你的电脑具备编程条件
2. **命令行基础** -- 学会用键盘控制电脑，这是所有开发者的基本功
3. **Markdown** -- 最简单的标记语言，5 分钟就能上手，用来记笔记和写文档
4. **HTML5** -- 网页的骨架，理解浏览器如何展示内容
5. **CSS** -- 网页的皮肤，让页面变得好看
6. **JavaScript** -- 网页的行为，让页面能够交互

### 有一些基础

如果你已经写过一些代码，但感觉知识零散：

1. **Git** -- 版本控制是协作开发的基础，越早学越好
2. **GitHub** -- 全球最大的代码托管平台，也是你的技术简历
3. **TypeScript** -- 给 JavaScript 加上类型系统，大幅减少低级错误
4. **选择一个方向深入** -- 前端（Vue/React）、后端（Python/Go/Java）、或数据（SQL/数据分析）

### 想系统建立知识体系

如果你希望构建完整的计算机知识框架：

1. **高等数学 + 线性代数 + 概率论** -- 数学是计算机科学的基础语言
2. **离散数学** -- 逻辑、集合、图论，直接服务于算法和数据库
3. **算法与数据结构** -- 决定你写出的代码是"能用"还是"高效"
4. **计算机基础** -- 操作系统、计算机网络、编译原理的核心概念
5. **软件工程** -- 从"能写代码"到"能做项目"的关键跨越

## 学习方法

### 渐进式学习

不要试图一次学完所有东西。有效的学习路径是：

- 先广后深 -- 先建立全局视野，再深入特定领域
- 实践驱动 -- 每学一个概念就动手写代码验证
- 间隔重复 -- 关键概念在不同阶段反复出现，逐步加深理解
- 项目串联 -- 用一个小项目把零散知识点连成体系

### 学习节奏

| 阶段   | 每日投入 | 持续时间 | 目标                         |
| ------ | -------- | -------- | ---------------------------- |
| 入门期 | 1-2 小时 | 2-4 周   | 掌握基础工具和一门语言语法   |
| 成长期 | 2-3 小时 | 2-3 月   | 能独立完成小项目             |
| 进阶期 | 2-4 小时 | 3-6 月   | 理解底层原理，能解决复杂问题 |

### 常见问题

**"我该学哪门语言?"**

取决于你的目标：

| 目标     | 推荐语言   | 原因               |
| -------- | ---------- | ------------------ |
| Web 前端 | JavaScript | 前端唯一语言       |
| 数据分析 | Python     | 生态完善，语法简洁 |
| 系统编程 | C/C++      | 底层控制，性能极致 |
| 企业开发 | Java       | 岗位多，生态成熟   |
| 游戏开发 | C#/C++     | Unity/Unreal 引擎  |

**"学不会怎么办?"**

- 降低难度：回到上一个理解的知识点
- 换个方式：看视频教程替代纯文字
- 找人讨论：在 GitHub Issues 或社区提问
- 休息一下：睡一觉，大脑会在后台处理

**"学了就忘怎么办?"**

遗忘是正常的。应对方法：

1. 间隔复习 -- 第 1 天、第 3 天、第 7 天各回顾一次
2. 费曼技巧 -- 用自己的话解释给别人听
3. 项目驱动 -- 做一个小项目，把知识点串起来
4. 标记进度 -- 用本站进度系统标记已读文档

## 心态建议

- 编程是一项技能，不是天赋。任何人都可以通过练习掌握
- 报错是正常的，专业开发者每天也在调试错误
- 不要和别人比速度，按自己的节奏前进
- 完成比完美重要，先让代码跑起来再优化
- 遇到困难时，记住：你已经比昨天的自己进步了
