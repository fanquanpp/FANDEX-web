## 1. 级联复制概述

级联复制允许备库作为其他备库的上游，减少主库的复制负载。

```
主库 → 级联备库1 → 备库2
                  → 备库3
```

## 2. 配置

```ini
# 级联备库配置（既是从库也是上游）
# postgresql.conf
hot_standby = on
max_wal_senders = 10    # 允许向下游发送WAL
wal_keep_size = '1GB'
```

```ini
# 下游备库的 primary_conninfo 指向级联备库
primary_conninfo = 'host=cascade-standby user=replicator password=password'
```

## 3. 使用场景

- 跨数据中心复制：主库→同城级联备库→异地备库
- 减少主库负载：多个备库从级联备库复制
- 报表分流：级联备库承担报表查询，同时向下游复制
