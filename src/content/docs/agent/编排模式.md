# 编排模式 — 监督者、群集、层级

> 四种编排模式在2026年框架中反复出现：监督者-工作者、群集/对等、层级、辩论。Anthropic的指导："关键在于为你的需求构建正确的系统。"从简单开始；仅在单个Agent加五种工作流模式不够时才添加拓扑。

**类型：** 学习 + 构建
**语言：** Python（标准库）
**前置条件：** Phase 14 · 12（工作流模式），Phase 14 · 25（多Agent辩论）
**时间：** ~60分钟

## 学习目标

- 命名四种反复出现的编排模式以及每种适合的场景。
- 描述2026年LangChain推荐：基于工具调用的监督vs监督库。
- 解释Anthropic的"构建正确系统"规则以及它如何门控拓扑选择。
- 用标准库针对通用脚本LLM实现所有四种模式。

## 问题所在

团队在需要之前就伸手要"多Agent"。四种模式在框架中反复出现；一旦你能命名它们，你就可以选择正确的 — 或完全跳过拓扑。

## 核心概念

### 监督者-工作者

- 一个中央路由LLM分派给专业Agent。
- 决策：循环回自身、移交给专家、终止。
- 专家之间不互相通信；所有路由通过监督者。

框架：LangGraph `create_supervisor`、Anthropic orchestrator-workers、CrewAI层级流程。

**2026年LangChain推荐：** 通过直接工具调用进行监督，而不是`create_supervisor`。提供更精细的上下文工程控制 — 你精确决定每个专家看到什么。

### 群集/对等

- Agent通过共享工具表面直接移交。
- 无中央路由器。
- 比监督者延迟更低（更少跳数）。
- 更难推理（无单一控制点）。

框架：LangGraph群集拓扑、OpenAI Agents SDK handoffs（当所有Agent可以移交给所有其他Agent时）。

### 层级

- 监督者管理子监督者，子监督者管理工作者。
- 在LangGraph中实现为嵌套子图；在CrewAI中为嵌套crew。
- 以运营复杂性为代价扩展到大型Agent群体。

何时需要：当单个监督者的上下文预算无法容纳所有专家的描述时。

### 辩论

- 并行提议者 + 迭代交叉批判（第25课）。
- 不真正是编排 — 更像是验证 — 但在框架中作为拓扑选择出现。

### CrewAI Crew vs Flow

CrewAI形式化了两种部署模式：

- **Flow**用于确定性事件驱动自动化（推荐的生产起点）。
- **Crew**用于自主基于角色的协作。

这与上述四种模式正交，但映射到拓扑：Flow通常是监督者或层级；Crew通常是带LLM路由器的监督者。

### Anthropic的指导

"在LLM领域的成功不在于构建最复杂的系统。而在于为你的需求构建正确的系统。"

决策顺序：

1. 单个Agent + 工作流模式（第12课）— 从这里开始。
2. 监督者-工作者 — 当你有2-4个专家时。
3. 群集 — 当延迟比推理清晰度更重要时。
4. 层级 — 仅当监督者上下文预算失败时。
5. 辩论 — 当准确性比成本更重要时。

### 这种模式出错的地方

- **拓扑优先思维。** 在确定多Agent解决什么问题之前就说"我们需要多Agent"。
- **群集中的弹跳移交。** A -> B -> A -> B。使用跳数计数器。
- **假层级。** 三层因为"企业"；两个实际团队。坍缩它。

## 构建

`code/main.py`用标准库针对脚本LLM实现所有四种模式：

- `Supervisor` — 中央路由器。
- `Swarm` — 带直接移交的对等。
- `Hierarchical` — 监督者的监督者。
- `Debate` — 并行提议者 + 批判。

每种模式处理相同的三意图任务（退款/缺陷/销售）。追踪形状不同。

运行：

```
python3 code/main.py
```

输出：每种模式的追踪 + 操作计数。监督者最清晰；群集最短；层级最深；辩论最昂贵。

## 使用

- **LangGraph**用于监督者和层级（嵌套子图）。
- **OpenAI Agents SDK**用于工具即移交（监督者形状）。
- **CrewAI Flow**用于生产确定性。
- **自定义**用于辩论或当你想要精确控制时。

## 练习

1. 通过移除路由器将监督者-工作者转换为群集。什么出问题？什么改善？
2. 为群集添加跳数计数器：3次移交后拒绝。它是否捕获A->B->A弹跳？
3. 为12专家领域构建两级层级系统。没有嵌套时上下文预算在哪里失败？
4. 在生产形状工作负载上分析四种模式。哪个在哪个指标上获胜（延迟、成本、准确性、可调试性）？
5. 阅读Anthropic的"Building Effective Agents"文章。将你的每个生产流程映射到四种之一。有没有不能干净映射的？

## 关键术语

| 术语               | 人们怎么说       | 实际含义                               |
| ------------------ | ---------------- | -------------------------------------- |
| 监督者-工作者      | "路由器+专家"    | 中央LLM分派给专家；他们不互相通信      |
| 群集               | "对等"           | 通过共享工具直接移交；无中央路由器     |
| 层级               | "监督者的监督者" | 用于大型群体的嵌套子图                 |
| 辩论               | "提议者+批判"    | 并行提议者、交叉批判（第25课）         |
| 基于工具调用的监督 | "无库监督者"     | 通过直接工具调用实现监督者以控制上下文 |
| Crew               | "自主团队"       | CrewAI的基于角色协作模式               |
| Flow               | "确定性工作流"   | CrewAI的事件驱动生产模式               |
