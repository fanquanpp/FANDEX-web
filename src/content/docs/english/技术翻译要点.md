## 1. 技术翻译的特点

### 1.1 技术翻译 vs. 一般翻译

| 维度   | 技术翻译      | 一般翻译   |
| ------ | ------------- | ---------- |
| 准确性 | 极高要求      | 较高要求   |
| 术语   | 专业且一致    | 可灵活处理 |
| 风格   | 客观、简洁    | 可有文学性 |
| 受众   | 专业人员      | 普通读者   |
| 时效性 | 要求快速      | 相对宽松   |
| 格式   | 保留代码/格式 | 自由排版   |

### 1.2 技术翻译的核心原则

| 原则       | 说明                             |
| ---------- | -------------------------------- |
| 准确性第一 | 技术内容翻译错误可能导致严重后果 |
| 术语一致   | 同一术语全文统一翻译             |
| 保留原文   | 代码、命令、路径不翻译           |
| 用户友好   | 译文应便于目标读者理解           |
| 遵循规范   | 遵循行业翻译规范                 |

## 2. 术语一致性 (Terminology Consistency)

### 2.1 术语管理的重要性

| 问题               | 后果               |
| ------------------ | ------------------ |
| 同一术语多种翻译   | 读者困惑，理解困难 |
| 不同术语同一翻译   | 无法区分概念       |
| 翻译与行业惯例不符 | 专业性受质疑       |

### 2.2 术语表 (Glossary)

建立并维护术语表是确保一致性的关键：

| 英文       | 推荐翻译 | 不推荐翻译 | 说明                |
| ---------- | -------- | ---------- | ------------------- |
| algorithm  | 算法     | 算术       | 不同于 arithmetic   |
| array      | 数组     | 阵列       | 编程语境            |
| buffer     | 缓冲区   | 缓存       | 不同于 cache        |
| cache      | 缓存     | 高速缓存   | 简洁优先            |
| class      | 类       | 类别       | OOP 语境            |
| compile    | 编译     | 编纂       | 编程语境            |
| component  | 组件     | 部件       | UI 语境             |
| debug      | 调试     | 除错       | 行业惯例            |
| deploy     | 部署     | 配置       | 不同于 configure    |
| framework  | 框架     | 架构       | 不同于 architecture |
| interface  | 接口     | 界面       | API 语境用"接口"    |
| interface  | 界面     | 接口       | UI 语境用"界面"     |
| library    | 库       | 图书馆     | 编程语境            |
| middleware | 中间件   | 中介软件   | 行业惯例            |
| package    | 包       | 包裹       | 编程语境            |
| repository | 仓库     | 存储库     | Git 语境            |
| runtime    | 运行时   | 运行时间   | 技术语境            |
| server     | 服务器   | 服务端     | 通用                |
| thread     | 线程     | 线         | 编程语境            |

### 2.3 术语翻译策略

| 策略      | 适用场景       | 示例                 |
| --------- | -------------- | -------------------- |
| 直译      | 有明确对应中文 | database → 数据库    |
| 音译      | 无对应中文     | GitHub → GitHub      |
| 音译+意译 | 部分可译       | Docker → Docker 容器 |
| 不译      | 约定俗成       | API → API            |
| 创造新词  | 全新概念       | blockchain → 区块链  |

### 2.4 不翻译的术语

| 类型     | 示例                     | 原因     |
| -------- | ------------------------ | -------- |
| 编程语言 | Python, Java, Rust       | 专有名词 |
| 框架/库  | React, Vue, Django       | 专有名词 |
| 工具     | Git, Docker, Kubernetes  | 专有名词 |
| 协议     | HTTP, TCP, WebSocket     | 缩写     |
| 命令     | npm install, git push    | 代码     |
| 文件名   | package.json, Dockerfile | 代码     |
| 缩写     | API, SDK, IDE            | 行业标准 |

## 3. 本地化策略 (Localization)

### 3.1 本地化的层次

| 层次 | 内容           | 示例                       |
| ---- | -------------- | -------------------------- |
| 语言 | 翻译文本       | button → 按钮              |
| 格式 | 数字/日期/货币 | 1,000.00 → 1.000,00 (德语) |
| 文化 | 文化适应       | 红色→喜庆(中国)/警告(西方) |
| 法律 | 法律合规       | 隐私政策、数据保护         |
| 技术 | 技术适配       | 字符编码、文本方向         |

### 3.2 中文本地化要点

| 要点   | 说明                   | 示例                 |
| ------ | ---------------------- | -------------------- |
| 简繁体 | 区分简体中文和繁体中文 | 软件简体/軟體繁體    |
| 标点   | 使用中文标点           | ，。！？而非 , . ! ? |
| 空格   | 中英文之间加空格       | 使用 Python 编程     |
| 量词   | 添加适当的量词         | 一个数组、一条记录   |
| 敬语   | 适当使用敬语           | 您（而非你）         |

### 3.3 UI 文本本地化

| 英文    | 中文   | 注意事项               |
| ------- | ------ | ---------------------- |
| Submit  | 提交   | 统一用"提交"而非"递交" |
| Cancel  | 取消   | 统一用"取消"而非"放弃" |
| Save    | 保存   | 统一用"保存"而非"储存" |
| Delete  | 删除   | 统一用"删除"而非"移除" |
| Edit    | 编辑   | —                      |
| Search  | 搜索   | 统一用"搜索"而非"搜寻" |
| Loading | 加载中 | 加"中"表示进行时       |
| Error   | 错误   | —                      |
| Warning | 警告   | —                      |
| Success | 成功   | —                      |

```

```

## 5. 技术文档翻译规范

### 5.1 代码与命令

| 规范         | 说明             | 示例                     |
| ------------ | ---------------- | ------------------------ |
| 不翻译代码   | 代码保持原文     | `console.log("hello")`   |
| 不翻译命令   | 命令行保持原文   | `npm install express`    |
| 不翻译路径   | 文件路径保持原文 | `/usr/local/bin/`        |
| 不翻译变量名 | 变量名保持原文   | `const userName = "..."` |
| 翻译注释     | 代码注释可翻译   | `// 初始化配置`          |

### 5.2 格式保留

| 元素          | 处理方式                   |
| ------------- | -------------------------- |
| Markdown 格式 | 保留格式标记               |
| HTML 标签     | 不翻译标签，翻译标签内文本 |
| 链接          | 保留 URL，翻译链接文本     |
| 表格          | 保留表格结构               |
| 代码块        | 不翻译代码块内容           |

### 5.3 标点规范

| 规范             | 说明             | 示例            |
| ---------------- | ---------------- | --------------- |
| 中文用中文标点   | 正文用中文标点   | 这是一个数组。  |
| 代码用英文标点   | 代码内用英文标点 | `arr[0]`        |
| 中英文之间加空格 | 提高可读性       | 使用 React 框架 |
| 数字与中文加空格 | 提高可读性       | 版本 3.0 发布   |

## 6. 质量保证 (Quality Assurance)

### 6.1 翻译质量评估

| 维度       | 评估标准             | 权重 |
| ---------- | -------------------- | ---- |
| 准确性     | 是否正确传达原文含义 | 40%  |
| 流畅性     | 是否符合目标语言习惯 | 25%  |
| 术语一致性 | 术语翻译是否统一     | 20%  |
| 格式规范   | 格式是否符合要求     | 10%  |
| 风格统一   | 风格是否前后一致     | 5%   |

### 6.2 质量检查清单

| 检查项     | 说明                       |
| ---------- | -------------------------- |
| 术语一致性 | 同一术语是否统一翻译       |
| 遗漏检查   | 是否有未翻译的段落         |
| 格式检查   | Markdown/HTML 格式是否正确 |
| 链接检查   | 链接是否有效               |
| 代码检查   | 代码是否保持原文           |
| 标点检查   | 标点使用是否正确           |
| 空格检查   | 中英文之间是否有空格       |
| 数字检查   | 数字格式是否正确           |

### 6.3 常见错误

| 错误类型   | 示例                      | 修正            |
| ---------- | ------------------------- | --------------- |
| 术语不一致 | cache → 缓存/高速缓存     | 统一为"缓存"    |
| 代码被翻译 | `npm install` → npm 安装  | 保持原文        |
| 格式丢失   | **bold** → bold           | 保留加粗标记    |
| 链接断裂   | [text](url) → [文本](url) | 检查 URL 有效性 |
| 标点错误   | 这是一个array。           | 这是一个数组。  |

## 7. 技术翻译工作流

### 7.1 标准工作流

```
1. 项目分析 → 确定范围、术语、风格
2. 术语准备 → 建立术语表
3. 翻译执行 → 逐段翻译
4. 自审校对 → 译者自检
5. 同行评审 → 他人审校
6. 质量检查 → QA 工具检查
7. 终审发布 → 最终确认
```

### 7.2 持续本地化

对于持续更新的项目（如开源软件文档），采用持续本地化流程：

```
源文档更新 → 自动检测变更 → 增量翻译 → 评审 → 发布
```

| 工具      | 功能              |
| --------- | ----------------- |
| Crowdin   | 自动同步 Git 仓库 |
| Transifex | Webhook 触发翻译  |
| Phrase    | CI/CD 集成        |
