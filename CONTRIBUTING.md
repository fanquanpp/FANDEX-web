# 贡献指南

感谢你对 FANDEX 项目的关注。本指南旨在帮助你了解如何为本项目贡献内容。

## 内容来源说明

本仓库所有内容均由人工与人工智能（AI）共同编写、搜集、整理与编排。我们欢迎任何形式的贡献，包括但不限于：

- 修正文档中的错误或不准确之处
- 补充遗漏的知识点或示例
- 更新过时的技术信息
- 改进文档结构与表述
- 新增模块或文档

## 内容质量要求

### 准确性

- 所有技术内容应基于官方文档或权威资料
- 代码示例应经过验证，确保可运行
- 引用第三方信息时，应注明来源

### 零基础友好

- 文档应面向零基础学习者，避免假设读者已有先修知识
- 首次出现的专业术语应给出简要解释
- 代码示例应包含中文注释，并附带预期输出
- 每篇文档应包含"常见错误与排查"章节

### 排版规范

- 中英文之间必须加半角空格
- 中文与数字之间必须加半角空格
- 统一使用中文标点（代码和链接除外）
- 不使用 emoji
- 标题层级不超过三级
- 代码块必须标注语言类型

### 术语统一

- Agent：首次出现写"智能体（Agent）"，后续统一用"智能体"
- API：统一保留英文缩写
- Gateway：统一使用"网关"

## 撰写内容要求

### 文字表述

- 以文字讲解说明为主，力求详细、清晰、易懂
- 避免过度依赖 Markdown 表格展示技术概念或操作步骤
- 表格仅适用于纯数据对比场景（如参数对照、版本差异等）
- 技术说明和操作步骤应使用段落文字配合代码示例展示
- 可使用 Mermaid 图表展示流程、架构、关系等可视化内容

### 禁止事项

- 禁止使用 emoji（包括 Unicode emoji 和短代码形式）
- 禁止使用 ASCII 艺术字（如文本框图、字符画等）
- 禁止在标题中使用特殊符号装饰
- 禁止使用纯大写字母表示强调（应使用加粗或斜体）

### 代码示例

- 代码块必须标注语言类型（如 sql、bash、python）
- 关键行必须有中文注释
- 非简单代码块应附带预期输出
- 提供完整可运行示例，避免仅展示片段
- 正确示例与错误示例应明确区分并标注

### 结构规范

- 文档应遵循统一结构：学习目标 → 前置知识 → 核心概念 → 动手实践 → 常见错误与排查 → 自测题 → 下一步
- 每篇文档应包含至少一个常见错误的示例与解决方法
- 文档末尾应包含"下一步"章节，链接到相关进阶文档
- 标题层级不超过三级，H4 仅在极必要时使用

## AI 协作说明

本仓库内容由人工与 AI 共同编撰，贡献者在提交内容时应注意：

- AI 生成的内容应经过人工审核与验证
- 不应直接提交未经核实的 AI 输出
- AI 辅助编写的内容应确保技术准确性
- 鼓励使用 AI 工具提升效率，但最终质量由提交者负责

## 提交规范

- 提交信息使用简体中文
- 每次提交应聚焦于单一变更
- 提交前运行 `npm run build` 确保构建通过
- 提交前运行 `npm run lint:docs` 检查文档格式
- 推送后 GitHub Actions 构建约需 12 分钟，构建完成后需手动刷新浏览器页面方可查看最新版本

## 自学能力倡导

本仓库鼓励学习者在学习过程中培养以下能力：

- **自主检索**：学会使用搜索引擎、官方文档、AI 工具高效获取信息
- **交叉验证**：对获取的信息进行多来源比对，确认准确性
- **批判性思考**：不盲信任何单一信息来源，包括本仓库内容
- **持续学习**：技术发展迅速，保持学习与更新的习惯

## 问题反馈

如发现内容错误或有改进建议，欢迎通过以下方式反馈：

- 提交 GitHub Issue
- 提交 Pull Request
- 在相关文档页面留言

感谢你的贡献。
